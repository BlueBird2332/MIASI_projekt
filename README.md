# pylintool

ANTLR-based Python linter and formatter. Built as a university project.

## Features

### Whitespace checks (always on, auto-fixable with `--fix`)

| Code | Description | Auto-fix |
|---|---|---|
| W001 | Tab used for indentation | yes |
| W002 | Mixed tabs and spaces in indentation | yes |
| W003 | Trailing whitespace | yes |
| W004 | More than 2 consecutive blank lines | yes |
| W005 | Tab character used for inline alignment | yes |
| W006 | Indentation not a multiple of 4 spaces | — |
| W007 | No newline at end of file | yes |
| W008 | Unexpected / redundant indentation | — |

### Type checking (`--typecheck`)

| Code | Description |
|---|---|
| T001 | mypy error or warning (delegates to mypy) |

### Code-quality heuristics (`--heuristics`)

| Code | Description |
|---|---|
| H001 | Line too long (default: 120 chars) |
| H002 | Function body too long (default: 50 lines) |
| H003 | Missing docstring on function or class |
| H004 | Missing type annotation on parameter or return |
| H005 | Too many function arguments (default: 6) |
| H006 | File too long (default: 500 lines) |
| H007 | TODO / FIXME comment |
| H008 | `print()` call — use logging instead |
| H009 | Bare `except:` clause |
| H010 | Magic number — should be a named constant |

## Prerequisites

- **Python 3.11+**
- **Java 11+** (needed only once, to generate the ANTLR parser)
- **[uv](https://docs.astral.sh/uv/)** (recommended package manager)

Install uv if you don't have it:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh    # macOS / Linux / WSL
```

## Quick start

```bash
git clone <your-repo-url>
cd pylintool

# installs Python, deps, and generates the ANTLR parser
make setup
```

`make setup` runs `uv sync --all-extras` then `make generate`.

## Usage

```bash
# Lint a single file (whitespace checks only)
pylintool path/to/file.py

# Lint a whole directory recursively
pylintool src/

# Auto-fix whitespace issues in-place
pylintool src/ --fix

# Enable type checking (requires mypy)
pylintool src/ --typecheck

# Enable code-quality heuristics
pylintool src/ --heuristics

# All checks at once
pylintool src/ --all

# CI mode: exit 1 if any issue is found
pylintool src/ --check

# Only print the summary line, not individual issues
pylintool src/ --quiet

# Custom heuristic thresholds
pylintool src/ --heuristics --max-line-length 80 --max-function-lines 30 \
               --max-arguments 4 --max-file-lines 300
```

## How ANTLR is used

The grammar (`PyWhitespaceLexer.g4` + `PyWhitespaceParser.g4`) tokenises Python
source while **keeping whitespace as visible tokens** instead of discarding it.
The parser groups tokens into lines with explicit `leading_ws` and `trailing_ws`
sub-rules. Two complementary techniques are then applied:

**Parse-tree listener** (`_WhitespaceListener`): walks the tree and fires on each
`blankLine`, `contentLine`, and `lastLine` node. Handles W001–W004 and W006.

**Token-stream scans** (post-parse): iterate over `CommonTokenStream.tokens`
directly for checks that need lookahead across lines — W005 (inline tab: a WS
token containing `\t` that is not at column 0 and is followed by a non-WS token
on the same line) and W008 (over-indentation: maintains an indentation stack,
`expecting_indent` flag, and `paren_depth` counter to distinguish legitimate
block indentation from redundant indentation).

The heuristics checker uses regex on raw source lines rather than ANTLR's full
Python3 grammar, which is complex and unnecessary for line-level metrics.

## Architecture decisions

- **uv** for reproducible environments — no Docker needed.
- **ANTLR version pinning**: the tool (`antlr4-tools`) and the runtime
  (`antlr4-python3-runtime`) must use the same version. Default is **4.11.1**.
  To change it, update both entries in `pyproject.toml` and re-run `make generate`.
- Generated parser files are **git-ignored** and rebuilt via `make generate`.
- Type checking delegates to **mypy** rather than reimplementing it; if mypy is
  not installed, a single INFO issue is reported instead of crashing.
